üìà Statistical Significance Guidelines

Don't Stop the Test Early!

‚ö†Ô∏è Common Mistake: Seeing early trends after 3 days and declaring a winner.

Problem: Small sample sizes create random fluctuations. You'll pick the "lucky" variation, not the truly better one.

Minimum Requirements Before Calling a Winner

Requirement Threshold Why
Statistical Confidence 95% or higher < 5% chance the result is random luck
Minimum Conversions Per Variation 100+ conversions each Small sample sizes = unreliable data
Minimum Test Duration 2 weeks (14 days) Accounts for weekly traffic patterns (weekend vs weekday behavior)
Consistent Results Winner holds for 3+ days straight Prevents declaring winner during temporary spike

üîç Troubleshooting Guide

Problem: Both Variations Have Low Conversion Rates

Symptoms: Both variations converting < 5% (below var110 baseline)

Possible Causes:

  • Traffic quality issue (wrong audience from video)
  • Checkout page problem (not landing page)
  • Price sensitivity (too expensive for this audience)
  • Message mismatch from video to landing page

Diagnosis:

  • Check CTA click-through rate. High clicks + low conversions = checkout problem
  • Review video traffic source. Is it qualified traffic?
  • Compare bounce rate to var110. Much higher? Message mismatch.
  • Check FAQ expansion rate. High = users have unaddressed objections

Problem: High Bounce Rate (> 70%)

Symptoms: Users landing on page and immediately leaving

Possible Causes:

  • Message mismatch from Silva video to landing page
  • Price shock ("$299/year" vs expected "$1.50/day")
  • Slow page load time
  • Hero section unclear or unappealing

Solutions:

  • Test more prominent price clarification in hero section
  • Add "As seen in the video with Vishen" trust element to hero
  • Optimize page load speed (images, scripts)
  • A/B test different hero headlines

Problem: Users Not Scrolling to Comparison Table

Symptoms: Low scroll depth (< 50%), comparison table not viewed

Diagnosis:

  • Check scroll depth analytics
  • Review comparison_view event‚Äîis it firing?
  • Look at time on page‚Äîare users reading or bouncing quickly?

Solutions:

  • Add visual "scroll indicator" arrow below hero
  • Test sticky comparison table summary that follows user
  • Add teaser text in hero: "See why the subscription is the same price below ‚Üì"
  • For Var A: Move comparison even EARLIER (Section 1.5, right after hero)

Problem: No Clear Winner After 2 Weeks

Symptoms: Both variations performing nearly identically

What This Means:

  • Test variable (comparison placement) doesn't significantly impact this audience
  • Both approaches work equally well (or poorly)
  • Need to test a BIGGER difference

Next Steps:

  • Pick the variation with slightly better numbers (even if not statistically significant)
  • Test a more dramatic change (e.g., different pricing display, testimonial placement, guarantee prominence)
  • Review qualitative feedback (customer support questions, user testing)
