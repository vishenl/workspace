Progress Tracking Template

Interactive CSV Template for Outcomes Analysis

About This Template

This CSV template helps you track your progress as you analyze and simplify outcomes. Use it to record your decisions, notes, and ensure consistent processing of all outcomes.

Usage: Download the CSV file below and import it into Excel, Google Sheets, or Airtable to start tracking your analysis progress.

Template Structure

Outcome ID

The unique Airtable record ID for the outcome (e.g., rec001, rec002, etc.)

Outcome Name

The name/title of the outcome being analyzed

Program Count

Number of programs using this outcome

Meditation Count

Number of meditations using this outcome

Audio Count

Number of audios using this outcome

Total Usage

Sum of Program Count + Meditation Count + Audio Count

Recommendation

Initial recommendation: KEEP, DELETE, or MERGE

Primary Outcome (if merge)

If merging, the name of the primary outcome this should merge into

UPDATE TYPE

Final decision: blank (KEEP), "Deleted", or "Merged"

UPDATE NOTE

Detailed reasoning for the decision

Reviewed By

Name or initials of person who reviewed this outcome

Review Date

Date when the outcome was reviewed (YYYY-MM-DD format)

Notes

Any additional notes, context, or considerations

How to Use This Template

Step 1: Export Outcomes from Airtable

  1. Open the ü™ΩOutcomes table in Airtable
  2. Use the üîß by AOG view
  3. Export to CSV (ensure you include: Outcome ID, Outcome Name, Program Count, Meditation Count, Audio Count)

Step 2: Enhance with Template Columns

  1. Open the exported CSV in Excel or Google Sheets
  2. Add the additional columns from this template:
    • Total Usage (formula: Program + Meditation + Audio)
    • Recommendation (formula or manual)
    • Primary Outcome (if merge)
    • UPDATE TYPE
    • UPDATE NOTE
    • Reviewed By
    • Review Date
    • Notes

Step 3: Process Each Outcome

  1. Review each outcome using the decision tree from the manual guide
  2. Fill in the Recommendation column
  3. For merges, specify the Primary Outcome
  4. Complete UPDATE TYPE and UPDATE NOTE
  5. Add your initials and date
  6. Add any relevant notes

Step 4: Track Progress

  1. Use filters to see outcomes by status
  2. Count completed reviews vs. total outcomes
  3. Generate summary statistics (deletions, merges, keeps)
  4. Share with team for validation

Step 5: Apply Updates

  1. Once validated, use the template data to update Airtable
  2. Import UPDATE TYPE and UPDATE NOTE back into Airtable
  3. Or use the data to generate batch update scripts

Pro Tips

  • Color code rows: Use conditional formatting to highlight KEEP (green), DELETE (red), MERGE (blue)
  • Sort strategically: Sort by Total Usage (ascending) to tackle low-usage outcomes first
  • Use filters: Filter by Recommendation to work on one category at a time
  • Add formulas: Create automatic recommendations based on usage thresholds
  • Share for review: Use Google Sheets for collaborative review with stakeholders
  • Version control: Save dated versions as you progress (e.g., outcomes-analysis-2025-11-11.csv)

Example Formula for Auto-Recommendation

In Excel or Google Sheets, you can use this formula in the Recommendation column:

=IF(AND(C2=0,D2=0,E2=0),"DELETE: No usage",IF((C2+D2+E2)<=2,"DELETE: Low usage","REVIEW: Check for merges"))

Where C2, D2, E2 are Program Count, Meditation Count, and Audio Count respectively.

Sample Workflow

Morning Session (2 hours):

  1. Filter for outcomes with Recommendation = "DELETE: No usage"
  2. Review each one (spot check for strategic importance)
  3. Fill in UPDATE TYPE and UPDATE NOTE
  4. Mark as reviewed with date
  5. Expected output: 40-60 outcomes processed

Afternoon Session (2 hours):

  1. Filter for outcomes containing "stress" or "anxiety"
  2. Sort by Total Usage descending
  3. Identify primary (highest usage)
  4. Mark others for merge with reference to primary
  5. Repeat for 2-3 more semantic groups
  6. Expected output: 20-30 merge recommendations
